 Marketplace Revenue Analysis
 Описание проекта

Анализ данных маркетплейса с целью оценки продуктовых метрик, выявления проблем роста и формирования гипотез для увеличения выручки и удержания пользователей.

 Данные

~3000 транзакций

5 сущностей: пользователи, транзакции, продавцы, товары, города

 Ключевые метрики

GMV: 8,041,833

Conversion Rate: 51.3%

Average Order Value: ~5096

 Основные результаты

Более 48% транзакций не завершаются успешно (pending + error)

Выручка сильно сконцентрирована географически:

Москва: 52.5%

Санкт-Петербург: 41.8%

Топ-10 продавцов генерируют 64% всей выручки

Наблюдается высокая зависимость бизнеса от ограниченного числа продавцов и регионов

 Выводы

Продукт зависит от небольшого числа продавцов → риск концентрации

Основной рынок — Москва и СПб → слабая региональная диверсификация

Высокая доля незавершённых транзакций указывает на проблемы в воронке

Гипотезы

Улучшение процесса оформления заказа (снижение error/pending)

Расширение ассортимента и продавцов в регионах

Поддержка новых продавцов для снижения зависимости от топ-10

Оптимизация пользовательского пути для увеличения конверсии
